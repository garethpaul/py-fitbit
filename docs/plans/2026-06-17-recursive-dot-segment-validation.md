# Recursive Dot-Segment Validation

Status: Planned

## Context

`validate_api_call` rejects raw dot segments and segments that become `.` or
`..` after one percent-decoding pass. Double-encoded dot segments and encoded
slashes can still pass local validation, for example `%252e%252e` and
`%2e%2e%2fprofile.json`. An intermediary or origin that performs another decode
can reinterpret those accepted paths as traversal segments.

## Priorities

1. Reject dot segments exposed at any percent-decoding layer, including encoded
   path separators, before OAuth or HTTPS objects are created.
2. Preserve valid Fitbit API paths, query strings, credential-query rejection,
   fragment and whitespace guards, and all token-cache safety boundaries.
3. Add mutation-sensitive behavior, source, documentation, and completed-plan
   contracts so validation cannot regress to one-pass segment decoding.

## Implementation Units

### U1: Fully decode the protected resource path for validation

Files:

- `fitbit.py`

Approach:

- Add a small path-validation helper that checks each current path form for
  exact `.` or `..` segments, repeatedly percent-decodes the full path, and
  stops when decoding reaches a fixed point.
- Validate the decoded full path rather than decoding already-split segments so
  encoded `/` separators expose newly formed traversal segments.
- Keep the original, validated API call unchanged for OAuth signing and the
  actual request.

### U2: Prove pre-network rejection and preserve valid paths

Files:

- `tests/test_fitbit_oauth_request.py`
- `scripts/check_legacy_fitbit.py`

Approach:

- Extend the existing invalid-path table with double-encoded dot segments and
  encoded-separator traversal forms.
- Preserve assertions that no OAuth request or HTTPS connection is created.
- Add static contracts for fixed-point decoding and the new regression values.

### U3: Synchronize maintenance evidence

Files:

- `README.md`
- `SECURITY.md`
- `VISION.md`
- `CHANGES.md`
- `docs/plans/2026-06-17-recursive-dot-segment-validation.md`

Approach:

- Document recursive percent-decoding and encoded-separator rejection without
  claiming protection beyond Fitbit protected-resource path construction.
- Record completed implementation and actual verification results.

## Risks

- Repeated decoding must terminate on malformed or stable percent sequences.
- Valid percent-encoded non-traversal path data must remain accepted and must be
  sent without normalization.
