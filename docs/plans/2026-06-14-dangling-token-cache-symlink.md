# Reject Dangling Token Cache Symlinks Before Network Access

Status: Completed

## Context

`os.path.exists` follows symbolic links. A dangling `access_token.string`
symlink therefore appears absent and can send the client into the interactive
OAuth flow before the later cache write rejects the symlink.

## Objectives

- Include dangling access-token cache symlinks in the cache-existence branch so
  the existing read guard rejects them before network access.
- Prove dangling symlinks fail before OAuth request construction or HTTPS
  connection creation.
- Preserve regular-file cache reads, owner-only permissions, and interactive
  token creation.
- Add mutation-sensitive source, test, documentation, and completed-plan
  contracts.

## Scope Boundaries

- Do not change OAuth endpoints, signing, response parsing, dependencies, or
  hosted workflow coverage.
- Do not add credentials, live Fitbit requests, or generated bytecode.
