# Bytecode Free Verification

Status: Completed

## Context

The legacy Python 2 test path imports `fitbit.py`, which can leave a local
`fitbit.pyc` file behind. The file is ignored by git, but generated bytecode in
the working tree makes repository verification noisy and can hide stale local
state.

## Objectives

- Remove generated Python bytecode from the repository checkout.
- Keep the Python 2 test target from writing local bytecode.
- Extend the Python 3 safety checker to fail when `.pyc` files or
  `__pycache__` directories exist in the repository tree.
- Document the guard in README, VISION, and CHANGES.

## Work Completed

- Removed the generated `fitbit.pyc` file from the local checkout.
- Added `PYTHONDONTWRITEBYTECODE=1` to the Makefile test target.
- Set `sys.dont_write_bytecode = True` in the mocked Python 2 test harness.
- Added `__pycache__/` to `.gitignore`.
- Extended `scripts/check_legacy_fitbit.py` to reject local bytecode artifacts.
- Updated top-level documentation for bytecode-free verification.
